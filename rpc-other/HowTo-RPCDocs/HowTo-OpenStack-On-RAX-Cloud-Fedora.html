How-To: OpenStack on RAX Cloud Servers#

This how-to provides instructions for building the prerequisite infrastructure for installing multi-host OpenStack environments on RAX cloud servers using the official installation guide.

The infrastructure contains three OpenStack nodes operating on private (tenant) networks managed by a network services (jump/firewall) node. The latter protects the OpenStack nodes from direct internet access and provides internet connectivity to instances in the environment.

Overview#

  • Prepare RAX cloud environment.
  • Build and configure network services (jump/firewall) node.
  • Build and configure OpenStack nodes.
  • Install and configure OpenStack services with minor modifications to the official installation guide.

Network architecture#

Prepare cloud environment#

Keys

  • Upload a public key.

Networks

Create the following networks:

  1. net-osmgmt1

    • 10.1.11.0/24

  2. net-osint1

    • 10.1.12.0/24

  3. net-osext1

    • 10.1.13.0/24

Launch servers#

Network services node (hst-ns1)#

  1. Launch node:

    • OS: Fedora 21 PVHVM
    • Flavor: 1 GB Performance 1
    • Networks: PublicNet, ServiceNet

    Note

    Adding multiple tenant networks simultaneously yields inconsistent network interface device names. This guide adds one tenant network at a time as it becomes necessary. Also, changing tenant networks after configuration erases changes made in this guide.

  2. Access the node using the IP address assigned by RAX:

    # ssh root@119.9.24.240

  3. Update node.

    # yum update -y

  4. Install additional packages.

    # yum install ntp shorewall -y

  5. Reboot the node:

    # reboot now

  6. Add the net-osmgmt1 network to the node.

  7. Add the net-osext1 network to the node.

  8. Configure additional network interfaces.

  9. Shut down eth2:

    # ifdown eth2

  10. Edit /etc/sysconfig/network-scripts/ifcfg-eth2. Do not touch the HWADDR line, as this is determined by the system:

    # Label net-osmgmt1
DEVICE=eth2
BOOTPROTO=static
HWADDR=bc:76:4e:18:03:b8
IPADDR=10.1.11.1
NETMASK=255.255.255.0
ONBOOT=yes
NM_CONTROLLED=no

  11. Bring up eth2:

    # ifup eth2

  12. Shut down eth3:

    # ifdown eth3

  13. Edit /etc/sysconfig/network-scripts/ifcfg-eth3. Do not touch the HWADDR line, as this is determined by the system:

    # Label net-osext1
DEVICE=eth3
BOOTPROTO=static
HWADDR=bc:76:4e:18:03:c2
IPADDR=10.1.10.1
NETMASK=255.255.255.0
ONBOOT=yes
NM_CONTROLLED=no

  14. Bring up eth3:

    # ifup eth3

  15. Create ~/vxlan1.sh with the following content:

    #!/bin/bash

modprobe vxlan
ip link add vxlan1 type vxlan id 1 group 239.0.0.1 dev eth3 dstport 4789
ip addr add 10.1.13.1/24 brd 10.1.13.255 dev vxlan1

  16. Run the vxlan1.sh script:

    # bash -x ~/vxlan1.sh

    This script needs to be run every time the node boots.

  17. Configure the firewall service.

  18. Set the following options in the /etc/shorewall/shorewall.conf file:

    STARTUP_ENABLED=Yes
...
IP_FORWARDING=On

  19. Edit the /etc/shorewall/interfaces file:

    ext eth0 routefilter,tcpflags
rax eth1
osm1 eth2
ose1 eth3
os1t vxlan1

  20. Edit the /etc/shorewall/masq file:

    eth0 10.1.11.0/24
eth0 10.1.13.0/24

  21. Edit the /etc/shorewall/policy file:

    $FW all ACCEPT
ext all REJECT
rax all ACCEPT
osm1 all ACCEPT
ose1 all ACCEPT
os1t all ACCEPT

  22. Edit the /etc/shorewall/rules file:

    Ping/ACCEPT ext $FW
SSH/ACCEPT ext $FW
#DNAT ext osm1:10.1.11.11  tcp    www
#DNAT ext osm1:10.1.11.11  tcp    6080

    Note

    Uncomment the DNAT rules and restart Shorewall as necessary to enable remote access to the dashboard and instance consoles in the OpenStack environment.

  23. Edit the /etc/shorewall/zones file:

    fw firewall
ext ipv4
rax ipv4
osm1 ipv4
ose1 ipv4
os1t ipv4

  24. Start the firewall service:

    service shorewall start

  25. Edit /etc/resolv.conf and add the Google DNS servers:

    nameserver 8.8.8.8
nameserver 8.8.4.4

  26. Test network connectivity to the internet by pinging openstack.org:

    # ping openstack.org
PING openstack.org (162.242.140.107) 56(84) bytes of data.
64 bytes from 162.242.140.107: icmp_seq=1 ttl=50 time=181 ms
64 bytes from 162.242.140.107: icmp_seq=2 ttl=50 time=180 ms
...

  27. Generate an ssh key for accessing other nodes:

    # ssh-keygen -t rsa -b 2048 -C "ns1" -P "" -f .ssh/id_rsa

OpenStack controller node (hst-os1ctl1)#

  1. Launch node, removing all networks except the net-osmgmt1 network:

    • OS: Fedora 21 PVHVM
    • Flavor: 4 GB Performance 1
    • Networks: net-osmgmt1

  2. Using the IP address and root password assigned by RAX, run the following commands from hst-ns1 to access hst-os1ctl1 using ssh:

    # ssh-copy-id -i .ssh/id_rsa.pub root@10.1.11.2    # ssh root@10.1.11.2

    Note

    The node cannot access the internet without additional configuration.

  3. Configure network interfaces.

  4. Edit /etc/sysconfig/network-scripts/ifcfg-eth0. Do not touch the HWADDR line, as this is determined by the system:

    # Label net-osmgmt1
DEVICE=eth0
BOOTPROTO=static
HWADDR=bc:76:4e:18:03:b8
IPADDR=10.1.11.11
NETMASK=255.255.255.0
GATEWAY=10.1.11.1
ONBOOT=yes
NM_CONTROLLED=no

  5. Edit the /etc/hosts file:

    # hst-os1ctl1
10.1.11.11  hst-os1ctl1

# hst-os1net1
10.1.11.21  hst-os1net1

# hst-os1cpu1
10.1.11.31  hst-os1cpu1

    Note

    Comment out or remove any existing lines containing hst-os1ctl1.

  6. Edit /etc/resolv.conf and add the Google DNS servers:

    nameserver 8.8.8.8
nameserver 8.8.4.4

  7. Reboot the node:

    # reboot now

  8. Access the node from the network services node using the new IP address on the net-osmgmt1 network:

    # ssh root@10.1.11.11

  9. Test network connectivity to the internet by pinging openstack.org:

    # ping openstack.org
PING openstack.org (162.242.140.107) 56(84) bytes of data.
64 bytes from 162.242.140.107: icmp_seq=1 ttl=50 time=181 ms
64 bytes from 162.242.140.107: icmp_seq=2 ttl=50 time=180 ms
...

  10. Update the node:

    yum update -y

  11. Reboot the node:

    # reboot now

Install and configure OpenStack services#

  1. Use the OpenStack Installation Guide with the following changes:

  2. Configuring the basic environment on all nodes:

    • Skip the network configuration sections.
    • Use 10.1.11.1 (network services node) as the NTP server.

  3. Configuring the Compute service on the compute node:

    • Use qemu instead of kvm virtualization.

  4. Configuring the Networking service on the network node:

    • Add the vxlan1 interface as a port on the br-ex bridge.

  5. Creating initial networks.

    • Use the following command for the subnet on the external network:
    neutron subnet-create ext-net --name ext-subnet \
--allocation-pool start=10.1.13.101,end=10.1.13.200 \
--disable-dhcp --gateway 10.1.13.1 10.1.13.0/24

    Note

    After performing the initial tenant network creation procedure, try pinging 10.1.13.101 from the network services node.
