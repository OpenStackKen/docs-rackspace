How-To: RPC Docs with a fork#

Before you begin, ask Lana or Karin to grant you access to the rackerlabs organization on GitHub.

Working with a fork provides an extra layer of "insulation" between your work in progress and the main repo. You can create branches in the main repo if you really want to, but note that extra branches in the main repo can create clutter and require regular pruning.

Forking the repository#

The procedure here uses https as an example. You can use ssh as well.

  1. In your browser, navigate to the docs-rpc repository.

  2. Click on Fork. When prompted, select your user ID.

  3. When the forking is complete, open a Terminal window and clone your fork:

    $ git clone https://github.com/USERNAME/docs-rpc.git

  4. Switch to the new cloned folder.

    $ cd docs-rpc

  5. Set your upstream to rackerlabs/docs-rpc.

    $ git remote add upstream https://github.com/rackerlabs/docs-rpc.git

  6. Check your git remote setup.

    $ git remote -v

    You should have something like this:

    origin https://github.com/USERAME/docs-rpc.git (push)
origin https://github.com/USERNAME/docs-rpc.git (fetch)
upstream   https://github.com/rackerlabs/docs-rpc.git (fetch)
upstream   https://github.com/rackerlabs/docs-rpc.git (push)

Note

You can also set remotes to point to other users' forks. This is useful for reviewing pull requests and collaborating on a change. For instance, if you want to create a remote for user janedoe:

$ git remote add janedoe https://github.com/janedoe/docs-rpc.git

Updating your fork#

  1. Change into the docs-rpc directory.

  2. If required, commit or stash changes on your working branch and checkout master:

    $ git stash
$ git checkout master

  3. Fetch and merge the upstream code.

    $ git fetch upstream
$ git merge upstream/master

  4. Push the updated upstream code to your GitHub fork.

    $ git push origin master

    Note

    If you're feeling clever, you can cut and paste those commands into a single line or create a bash script that runs them all:

    $ git fetch upstream; git merge upstream/master; git push origin master

    #!/bin/bash

git fetch upstream
git merge upstream/master
git push origin master

Do not forget the git push origin master step. Otherwise your fork on GitHub will fall behind the upstream repo, even though your local repo is up to date.

Editing, branching, and pull requests#

Before you begin, make sure you have updated your fork.

  1. Create a branch off master in which to do your work. A good naming convention for a branch is an issue number or the topic of your work. For example: issue-14 or update-links.

    $ git checkout -b BRANCHNAME

  2. After you've finished your work locally, commit it. If it resolves an issue, be sure to include the issue number in the correct format.

    $ git add .
$ git commit -a

    Meaningful commit message on first line

Further explanation if needed

Fixes #14

  3. Push the change to your fork on GitHub.

    $ git push origin BRANCHNAME

  4. In your browser, go to your forked repository.

    $ open https://github.com/USERNAME/docs-rpc

  5. At the top of the screen you'll see a status bar showing the change you've just committed and a green button to compare and create a pull request. Verify that the base branch is correct, then follow the prompts to create the pull request.

  6. Ask someone else to review, test build, and merge your commit.

Creating a pull request against a version branch#

Sometimes you may need to make a change on a particular version branch that affects only that version. This is the procedure.

  1. Run the following commands to checkout the required stable branch. This example uses v12.

    $ git checkout v12
$ git fetch upstream
$ git merge upstream/v12
$ git push origin v12

  2. Create a working branch off the version staging branch:

    $ git checkout –b BRANCHNAME

  3. Make your changes and commit them.

  4. Push the changes to your fork:

    $ git push origin BRANCHNAME

  5. Go to the docs-rpc repo page on GitHub and create a pull request. On the Compare changes page, make sure you have the following options selected:

    * base fork: rackerlabs/docs-rpc
* base: v12
* head fork: USERNAME/docs-rpc
* compare: BRANCHNAME

  6. Complete the pull request as normal.
