How-To: Generate RPC documentation release notes#

The purpose of the documentation release notes is to inform stakeholders about changes in documentation between releases.

The process involves generating a changelog based on the RPC release tag.

  1. Check merged pull requests are assigned the release tag that matches the RPC engineering release tag. For more information, see How-To: Tag RPC GitHub commits.

  2. Clone the docs-rpc repository:

    $ git clone https://github.com/rackerlabs/docs-rpc.git

    Note

    Clone the upstream docs-rpc repository. The changelog generator will not work correctly from a personal fork.

  3. Create a new branch:

    $ git checkout -b NEWBRANCHNAME

  4. In your local docs-rpc directory, install the GitHub Changelog Generator:

    $ gem install github_changelog_generator
  5. Generate a token

    to allow more than 50 unauthenticated GitHub requests per hour.

  6. Set the environment variable:

    $ export CHANGELOG_GITHUB_TOKEN="«your-40-digit-github-token»"

    Note

    For your convenience, set this variable in your bashrc file.

  7. Create a .github_changelog_generator file to override the default parameters and configure your generated output. For example:

    header-label=# RPC Documentation Change Log
unreleased=false
issues=false
author=false
pr-label=### Documentation changes:

    For more information on parameters, see Advanced change log generation examples

  8. Run the changelog command to generate a changelog.md file:

    $ github_changelog_generator

  9. Edit the file to include only notable merged PRs. such as revisions and changes to technical information, migrated content, and changes to the content structure. Do not include editorial changes and changes to the documentation toolchain.

  10. Convert the changelog.md file to RST format using Pandoc:

    $ pandoc -s rst changelog.md -o changelog.rst

    Note

    Pandoc conversion is not perfect, so editing the file may be required to comply with RST conventions.

  11. Add the changelog.rst and .github_changelog_generator files to your commit:

    $ git add changelog.rst .github_changelog_generator

  12. Commit your changes:

    $ git commit

  13. Push your commit to the docs-rpc repository:

    $ git push -u origin NEWBRANCHNAME

  14. Go to the GitHub docs-rpc repository and create a pull request.
